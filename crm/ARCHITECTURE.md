# AI CRM — Архитектурная спецификация
## МеталлПортал → SaaS Платформа
*Версия 1.0 | Апрель 2026*

> Полный текст спецификации — в сообщении от 20 апр 2026.
> Ниже — ключевые принципы и план реализации.

## ПРИНЦИПЫ
1. ИИ делает — человек подтверждает (Decision Queue)
2. Прозрачность каждого шага (лог рассуждений)
3. Данные принадлежат клиенту (RLS изоляция)

## СТЕК
- **Frontend**: Next.js 16, TypeScript, Tailwind, Vercel
- **DB**: Supabase (PostgreSQL + RLS + Realtime)
- **Оркестратор**: n8n (self-hosted)
- **ИИ**: GPT-4o (решения), GPT-4o-mini (рутина), Whisper (звонки)
- **Голос**: ElevenLabs + Voximplant / Twilio
- **Мессенджеры**: Telegram Bot, WhatsApp WABA (360Dialog)
- **Email**: Resend.com
- **Трекинг**: собственный JS скрипт → Supabase

## ПЛАН PHASES

### ✅ Phase 1 — Ядро (СДЕЛАНО)
- Мульти-тенант БД (tenants, crm_users, contacts, deals, activities, ai_queue)
- Авторизация через admin_users (как на сайте)
- Дашборд, контакты, сделки (канбан), очередь ИИ, настройки
- Деплой: https://metallportal-crm2.vercel.app

### Phase 2 — Трекинг и шина
- JS трекинг скрипт (track.js) для сайта
- n8n оркестратор + webhook приёмник
- Связка: событие на сайте → карточка контакта

### Phase 3 — ИИ-мозг
- Scoring лидов (матрица 0-100)
- GPT-4o: анализ запросов, генерация КП и писем
- Decision Queue: ИИ предлагает → менеджер одобряет
- Уведомления в Telegram

### Phase 4 — Каналы
- Email (Resend)
- Telegram бот (входящие/исходящие)
- WhatsApp WABA
- SMS (СМС-центр)

### Phase 5 — Телефония
- Voximplant интеграция
- Whisper транскрипция звонков
- ElevenLabs голос для ИИ-звонков
- Warm transfer (ИИ → менеджер)

### Phase 6 — Аналитика
- Дашборд руководителя
- Ежедневные отчёты (9:00)
- Анализ звонков + качество менеджеров
- Маркетинговые инсайты

### Phase 7 — Безопасность
- Детальные права доступа
- Обезличивание в аналитике (агрегаты ≥ 5 чел)
- Лог доступа к базе
- Алерты при массовом экспорте
